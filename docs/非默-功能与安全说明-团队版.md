## 一、现有功能

### 1. 紧急求助（SOS）
- 长按 2 秒触发 → 自动获取定位 → 向紧急联系人（最多 2 人，只存在用户手机里）发送预设求助短信 + 高德地图位置链接
- 登录墙右下角有隐蔽的 ‼️ 入口——不解锁也能触发 SOS
- 求助结束后推荐相关公益机构

### 2. 加密存证（核心功能）
- 拍照 / 录像 / 录音 / 导入已有文件，全部在手机上加密后才上传云端
- 取证瞬间自动封存时间、位置、设备信息；区分"现场取证"和"事后导入"两个等级
- 查看、导出、删除每个操作都要重新输入密码
- **导出举证包**：一键打包解密后的原始文件 + 双语举证说明（含哈希校验方法、报警/保护令/诉讼场景指引），不依赖非默 App 也能验证
- 删除有 72 小时冷静期（防止被施暴者胁迫删证），界面上看起来是立即删除
- 忘记密码：用注册时抄在纸上的 12 位恢复码找回

### 3. 心理 / 法律援助目录
- 按城市筛选（目前：全国热线 + 北京/上海/广州/深圳，省会城市扩充中）
- 覆盖家暴、性侵、骚扰、婚姻家事、心理支持、综合维权
- 每条资源标注"核实于 X年X月"；超过 12 个月未复核会显示黄色过期提醒
- 每周自动巡检：官方来源网页失效或号码从页面消失会自动报警

### 4. 账号与登录
- 邮箱验证码登录（同一设备之后只需输密码）
- 密码修改、语言切换（全程中英双语）、意见反馈

## 二、安全架构（说人话版）

**核心原则：服务器上只有乱码，钥匙永远在用户自己手里。**

1. **端上加密**：文件在用户手机上就加密完成（AES-256-GCM，银行级算法），服务器只收到密文
2. **密码不出设备**：密码只在手机本地"变出"解密钥匙，从不发送到任何服务器
3. **谁都解不开**：我们团队、Supabase 后台、黑进服务器的黑客、调证的执法机构——拿到的都是乱码。这不是承诺，是数学
4. **代价（刻意设计）**：用户把密码和纸质恢复码都弄丢 = 数据永久无法恢复，我们也帮不了
5. **紧急联系人、密码等个人信息只存在用户手机里**，服务器上没有
6. **2026-07 安全升级**：
   - 每个敏感操作（查看/导出/删除）都要重新输密码
   - 禁止弱密码（太短、纯数字、常见密码一律拒绝）
   - 密码防破解算法升级为 Argon2id（现行行业标准，让显卡暴力破解成本提高 10 倍以上；老用户下次解锁时自动无感迁移，迁移采用"先验证新锁能打开、再替换旧锁"策略，任何故障都不会导致用户被锁在门外）

**对外宣传口径（已确认可以说的）：**
> "服务器只存乱码，任何人——包括我们自己——都无法解密用户的证据。"

**对外不许说的（红线，Aspire News 前车之鉴）：**
- 不夸大安全性；时间戳存证服务目前是"接入中"，不能说"已上链"/"已公证"
- 不说"绝对安全""100% 防泄漏"——诚实是这个产品的生命线

## 三、诚实的现状评估（内部同步，2026-07 自评）

**综合安全评分：70/100。** 加密设计约 90 分（架构是真正的行业正确做法），失分在运营侧：

| 已知短板 | 状态 |
|---|---|
| 网页 App 每次打开都从服务器加载代码（理论上部署渠道被入侵可下发恶意代码） | 长期方案 = 原生 App（实体成立后） |
| 代码仓库尚未转私有 | **正在处理（2026-07-20）** |
| 没有第三方安全审计（"自己审自己"不够格） | 黑客松争取目标 |
| 浏览器历史会留下访问痕迹（对"身边人翻手机"的场景是暴露风险） | 待做：伪装/无痕方案 |
| 邮件验证码每小时限约 4 封（Supabase 免费版） | 真实用户到来前换自建邮件服务 |

**数据泄漏风险排序（内部认知统一）：**
1. 服务器被黑导致内容泄漏 → 接近零（只有乱码）
2. 施暴者拿到解锁状态的手机 / 胁迫 → 最现实的风险，靠冷静期等设计减损
3. 用户弱密码 → 已通过强制密码强度 + Argon2id 大幅缓解
4. "她在用非默"这个事实泄漏（使用痕迹）→ 存在，隐蔽性设计持续加强中
